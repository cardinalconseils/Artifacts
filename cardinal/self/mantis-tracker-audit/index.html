SCAN
Phase 01

Record live browser traffic

MANTIS intercepts network requests live as the page loads and as you interact. The quality of your recording determines everything downstream.

Prerequisite
Clean profile
Chrome Guest mode or fresh profile. No cached cookies, consent states, or session tokens.
State 1
Pre-consent load
Let the page fully load without touching the cookie banner. The legally critical window.
State 2
Post-consent interaction
Accept the banner, navigate 2–3 pages, trigger cart / signup / contact if relevant.
Output
Session with N requests
Typical sites: 150–400 total requests. Complex marketing stacks: 400+.
CLASSIFY
Phase 02

Cross-reference against Tracker Radar

MANTIS uses DuckDuckGo's Tracker Radar — the same classification dataset that powers the DDG browser — to identify every request by vendor, product, role, and legal category.

LOW — no active privacy signals detected
MEDIUM — behavioral or cross-site signals
HIGH — fingerprinting or aggressive tracking

Confidence 95%+ = treat as confirmed. Below 95% = verify manually.

TRACE
Phase 03

Follow upstream & downstream call chains

Use the Upstream / Downstream toggle in MANTIS to trace each vendor's execution context.

↑

Upstream — who loaded this?

Identifies the initiating script. tag_manager entries (GTM) are usually the root of most chains. If GTM fires pre-consent, every downstream vendor also fires pre-consent.

↓

Downstream — what does it load next?

Maps the vendor's call chain. A high downstream count (10+) means one consent failure cascades into many tracker fires. Priority for remediation.

?

Unclassified — investigate manually

Domains not in Tracker Radar DB. May be custom pixels, subdomain proxies, or obscure analytics. Never skip this cluster.

DECIDE
Phase 04

One decision per vendor — no exceptions

Every vendor in the findings table gets a decision before you close the session.

Keep

Core functionality

Low risk, consent-gated. Document and move on.

Flag

High downstream / unclear basis

Escalate to legal or DPO review before next deploy.

Remove

No consent, no legitimate interest

Tag for GTM or developer removal this sprint.

Consent-gate

Useful, needs explicit opt-in

Ensure CMP blocks it until user accepts. Verify in pre-consent recording.

What you get

Complete tracker inventory in under 30 minutes.

MANTIS + DuckDuckGo Tracker Radar gives you vendor identification, execution flow mapping, privacy risk scoring, and a findings file with clear actions — without touching the site's code or needing server access.

Where this applies

Any public web page

Client sites, Cardinal Conseils properties, PayFacto web layer, competitor marketing stacks, government portals for benchmarking. Anything a browser can load.

The legal angle

Loi 25 · PIPEDA · GDPR

The pre-consent recording state reveals what fires before a user consents. Every vendor in that window without a legitimate interest exemption is a compliance gap.

The CISO angle

Web-layer input for threat model

Third-party trackers are also third-party code execution on your domain. Advertising pixels with high downstream counts expand the attack surface for supply-chain compromise.

Proof of implementation

You know it worked when:

  • ✓Findings table complete — DECIDE column filled for every vendor
  • ✓Both pre-consent and post-consent states recorded
  • ✓At least one pre-consent vendor identified (or confirmed zero)
  • ✓Unclassified cluster is empty or each entry is explained
  • ✓GTM's full downstream chain traced — which tags, under what conditions
  • ✓Written action list with owner and timeline attached
  • ⚠NOT done if: only one recording state was captured
  • ⚠NOT done if: Unclassified cluster was skipped without investigation
  • ⚠NOT done if: any vendor row has no DECIDE value

Step by step

Implementation Workflow

  • 1

    Install MANTIS and prep the environment

    Install MANTIS Chrome extension. Open Chrome Guest mode or a fresh profile with no cookies, sessions, or other extensions active.

    Produces: clean recording environment5 min (first install)
  • 2

    Record State 1 — pre-consent

    Navigate to the target URL. Start MANTIS recording. Let the page fully load without interacting with any banner or CTA. Note the request count at stabilization.

    Produces: pre-consent session2–3 minLegally critical window
  • 3

    Record State 2 — post-consent interaction

    Accept the cookie banner. Navigate 2–3 key pages. Trigger any cart, signup, or contact flow. Let MANTIS capture the full interaction state.

    Produces: full interaction session5–10 min
  • 4

    Navigate the vendor graph

    Expand each cluster in MANTIS graph view. For each vendor: check Classification, Privacy Risk badge, and Execution Flow. Use Upstream/Downstream to trace call chains. Pay special attention to tag_manager entries and the Unclassified cluster.

    Produces: vendor map10–15 min
  • 5

    Export and fill the findings file

    Click Export → pull the request log. Open the findings template. Fill one row per vendor: name, product, role, legal category, privacy risk, downstream count, total endpoints.

    Produces: vendor inventory table5–10 min
  • 6

    Apply DECIDE logic

    For each vendor: Keep / Flag / Remove / Consent-gate. No row left blank. Add the three Loi 25 compliance columns if this is a formal audit: Requires Consent, Currently Gated, Gap.

    Produces: action list5 minSession complete

What kills the audit

Pitfalls & Failure Modes

!Recording in your main browser profile▶
Cached consent states and session cookies suppress tracker fires that would hit a real new visitor. Your session looks clean. A first-time visitor's session fires everything. Always use Guest mode or a fresh profile — this is the most common audit error.
!Stopping before user interaction▶
Many trackers don't fire on page load — they fire on scroll, click, or form fill. A recording that stops after the initial load misses conversion pixels, retargeting fires, and chat widget initialization. Record at least one complete user journey.
!Ignoring the Unclassified cluster▶
Unclassified doesn't mean harmless — it means MANTIS has no Tracker Radar entry for that domain. These are the entries worth investigating manually. They could be custom pixels, obscure analytics platforms, or first-party subdomains proxying third-party data. Never skip this cluster.
!Treating tag_manager as "just infrastructure"▶
GTM is the master loader. In a typical site it initiates 14+ downstream requests. If GTM fires before consent, everything it loads also fires before consent — regardless of whether each individual tag has a consent trigger configured. GTM consent triggering is where most Loi 25 violations actually live. Always trace GTM's full downstream chain in the pre-consent state.
!Confusing Privacy Risk LOW with "no problem"▶
LOW risk means no active privacy signals were detected in your current recording session — not that the vendor is inherently safe or compliant. A DoubleClick tag showing LOW risk can still be a Loi 25 violation if it fires pre-consent. Risk badge describes signal type in this session; compliance status depends on consent timing.
!One-state recording▶
Recording only after consent accept gives a false-clean picture. The legally dangerous window is the pre-consent state — what fires between page load and the user touching the consent banner. Always document both states, and specifically call out what vendors appear in the pre-consent list that are not legitimately interest-exempt.
!Leaving the session without a DECIDE per vendor▶
A findings file with no DECIDE column is observation, not audit. The value of MANTIS is not the vendor list — it's the action list. Every vendor row must have Keep / Flag / Remove / Consent-gate before the session is closed. If you can't decide in the session, it defaults to Flag.
