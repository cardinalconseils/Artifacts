B
Brief

Lock 5 decisions before writing

No writing happens until five decisions are locked. These are not preferences — they change the document structure, appended sections, and publishing path.

Decision 1
Mode
self / client / product / framework — multi-select OK. Changes what’s appended after the ten-part structure.
Decision 2
Venture
cardinal / payfacto / serviconnect / proposai / personal. Drives Obsidian folder placement.
Decision 3
Format
md (always) + interactive-html / simple-html / json / docx. MD is never optional.
Decision 4
Branding
cardinal-brand or unbranded. Affects voice, visual tokens on HTML output, and footer.
Decision 5
Publish
publish (GitHub + Vercel) or private (vault only). Suggested default: client/product/framework → publish; self → private.
U
Unpack

Extract mechanism, not summary

Silently accesses source material via web fetch, Brain search, or pasted content. Extracts the underlying logic — not a summary.

  • The promise — what the source claims you’ll get
  • The mechanism — why it works (actual cause-and-effect)
  • Required inputs — what you need before starting
  • The sequence — what order things must happen
  • Failure modes — where this breaks specifically
  • Hidden assumptions — what the source takes for granted

If source is inaccessible

Skill asks you to paste the key content. It will not hallucinate mechanism or sequence from a vague description.

I
Install

Write the ten-part structure

Every guide uses the same skeleton. The headers don’t change. What fills them does — mode changes the content, not the structure. See the Guide Structure tab for all ten sections.

L
Layer

Append mode-specific sections

After the ten-part structure, appends one or more mode-specific sections in this order:

  • self → Sprint Plan (30-min sprints, ADHD-friendly pacing)
  • client → Delivery Kit (RACI, kickoff agenda, handoff checklist)
  • product → Commercialization Pack (positioning, pricing tiers, hooks, upsell)
  • framework → Teaching Kit (diagram, worked examples, IP defense note)

The skeleton

Ten-part guide structure

Every guide uses this skeleton. Click any section for details.

1The Promise▶
One paragraph. What the reader has after finishing. Concrete, time-bound, measurable. No “you’ll understand” — only “you’ll have built X / be able to do Y / have shipped Z.”
2The Strategy — Where We Play / How We Win / What We Do Better▶
The strategic spine. Uses the Lafley/Martin Playing to Win cascade. Three sub-parts: Where we play (specific enough to disqualify someone), How we win (given real constraints), What we do better (repeatable, not lucky). Failure mode: “We win on quality” is not a strategy. If it applies to any business anywhere, rewrite it.
3Who This Is For / Who It’s Not For▶
Two short lists. Qualification filter. Skipped in self mode. Product mode: buyer disqualifiers raise conversion. Framework mode: “this applies when…”
4The Core Framework▶
3–7 principles or stages. Each gets a name and a one-sentence explanation. Load-bearing IP — make it memorable. Collapse to an acronym or visual model if possible. Must operationalize the Strategy from Section 2. If there’s no causal link, one of them is wrong.
5The Implementation Path▶
Sequenced steps, not a checklist dump. Each step: Name (verb-led), What you do (1–3 sentences), What you produce (the artifact), How long (realistic). Order matters. Parallels called out explicitly.
6The Tools & Stack▶
What the reader needs to execute. Prefer PMC’s stack: Next.js, TypeScript, Supabase, Tailwind, n8n, Stripe, Telnyx, ElevenLabs, Deepgram, OpenAI, Gemini. Note free vs. paid. Flag technical setup requirements.
7Templates & Scripts▶
No assets = no guide. At least one copy-paste asset: email templates, prompts, config files, SQL schemas, n8n JSON, scripts, printable checklists. This is what makes the guide deployable instead of theoretical.
8Pitfalls & Failure Modes▶
What kills implementation. Be specific. “Don’t skip step 3” is useless. “If you skip the webhook signature verification, Stripe will silently accept spoofed events and your subscription state will desync from production” is useful.
9Proof of Implementation▶
How the reader knows it worked. Concrete signals — a metric, an artifact, a behavior change, a working endpoint. Distinguish between built it and it’s working.
10Next Level▶
What to do once it’s running. Keeps momentum after launch and creates the natural upsell/upgrade path for product and client modes.

vercel-deploy.yml
name: Vercel Production Deployment
on:
  push:
    branches: [master]
env:
  VERCEL_ORG_ID: ${{ secrets.VERCEL_ORG_ID }}
  VERCEL_PROJECT_ID: ${{ secrets.VERCEL_PROJECT_ID }}
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm install --global vercel@latest
      - run: vercel pull --yes --environment=production --token=${{ secrets.VERCEL_TOKEN }}
      - run: vercel build --prod --token=${{ secrets.VERCEL_TOKEN }}
      - run: vercel deploy --prebuilt --prod --token=${{ secrets.VERCEL_TOKEN }}

Publishing pipeline

From Claude Code to live URL

🔍

1. Check repo state

GitHub MCP reads current folder. Same slug + different guide → append -v2. Never overwrite silently.

📦

2. Single commit

Pushes README.md, index.html, meta.json, regenerated root README.md and _index.json. One commit per publish.

⚡

3. GitHub Actions fires

Push to master triggers vercel-deploy.yml: vercel pull → vercel build --prod → vercel deploy --prebuilt --prod.

✅

4. Live URL confirmed

Skill surfaces GitHub URL (immediate) and Vercel live URL (~60s). Reports building / live / errored status.

Required secrets

GitHub Actions → Settings → Secrets → Actions

VERCEL_TOKENPersonal access token from vercel.com/account/tokens
VERCEL_ORG_IDFrom vercel.json or run vercel env locally
VERCEL_PROJECT_IDFrom .vercel/project.json after running vercel link locally

Failure modes

Push fails (auth)
GitHub MCP session expired. Reconnect. Obsidian save already done — not lost.
Vercel build errors
Secrets not set or project not linked. Check the Actions tab for the specific error.
live_url 404
index.html missing from guide folder, or push landed on non-master branch.
Folder collision
Same slug, different guide. Skill auto-appends -v2. Say "update this" to overwrite in place.
