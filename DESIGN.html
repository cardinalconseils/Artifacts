§ 01 · Visual Theme & Atmosphere

Editorial magazine meets hospitality concierge.

Wide margins, a tall refined serif, deep ink on warm cream — with a single confident cardinal-red accent that earns its place.

Warm & editorial Paper not screen Calmly expert Senior judgment One accent. Used sparingly. No gradients No glassmorphism No emoji in product

"You stay the expert. We stay invisible."

— Brand posture, Cardinal Conseils

Cardinal sells to partners who sell to hospitality operators. The visual language respects that hierarchy — it never upstages the partner, never speaks down to a venue, never gets excited about technology for its own sake. Every design decision asks: does this feel like the FT, not like a SaaS blog?

§ 05 · Layout Principles

8px base. Generous whitespace.

Spacing follows an 8-point grid. Marketing pages target 96–128px between sections. Dashboards target 24–32px between modules. Cramming is a cardinal sin.

Pills (999px radius) are reserved for status badges where the shape carries semantic meaning. Card corners are small (4–12px). No fully-rounded cards.

Grid System

§ 06 · Depth & Elevation

Shadows are reserved. Warm-tinted.

Cards are flat. Elevation is whisper-light — only floating elements (menus, toasts, modals) get a soft shadow. All shadows are warm-tinted, never neutral gray.

§ 07 · Do's and Don'ts

Rules that protect the register.

Cardinal's editorial tone is fragile. One wrong decision pulls the brand toward generic SaaS. These rules are not aesthetic preferences — they are brand integrity.

Do

  • Use cardinal red (#8B1F2A) for exactly one focal element per view — one CTA, one key metric, one status pip
  • Use sentence case for all headings, buttons, and navigation labels
  • Use Geist semibold (600) with tight tracking for display text (28px+) and hero headlines
  • Maintain at least 96px vertical spacing between marketing sections
  • Use hairline borders (1px, rgba(26,23,20,0.12)) for card separation
  • Use Geist Mono only for IDs, system codes, metadata, and small labels
  • Use bronze (#A67C52) for pull-quotes, divider rules, and scorecard medals
  • Use the pill radius (999px) only on status badges where shape carries meaning
  • Use warm-tinted shadows (rgba(26,23,20,...)) — never neutral gray
  • Write "Phase 1 · Discover" with a middle-dot separator (not a colon)

Don't

  • Don't use gradient text or gradient buttons — ever
  • Don't use glassmorphism beyond the sticky-nav scroll case
  • Don't use emoji in product UI, marketing copy, or templates
  • Don't use a bluish-purple "AI" palette — Cardinal is hospitality first
  • Don't use rounded-pill cards with a single colored left border accent (AI slop tell)
  • Don't use more than one cardinal-red element per view
  • Don't use Geist Mono as a decorative choice — it is a functional signal only
  • Don't use auto-playing video heroes or floating testimonial carousels
  • Don't nest cards inside cards — use spacing and type hierarchy instead
  • Don't use all-caps for body copy or long passages — only eyebrow labels

§ 08 · Responsive Behavior

Mobile-first with editorial restraint.

Single-column below 768px. Navigation collapses. Typography scales with clamp(). Whitespace reduces proportionally — never eliminated.

Breakpoint Width Grid Container padding Notes
xs (mobile) 0–639px 1 column 20px Single column, nav collapses to active tab only
sm (tablet) 640–767px 2 columns 24px 2-col cards, stacked phase tracker
md (landscape) 768–1023px 2–3 columns 32px Full nav visible, hero at 48px
lg (desktop) 1024–1199px 3–4 columns 32px Full design intent at this breakpoint
xl (wide) 1200px+ Max 1200px 32px Content caps at --max-content: 1200px

Typography Clamping

display: clamp(48px, 6vw, 88px) h1: clamp(36px, 5.5vw, 56px) h2: clamp(28px, 4vw, 40px) body: 16px (fixed)

Sticky elements: top nav scrolls into position at 80px. In product, left sidebar fixed; single bottom-right support handle. Nothing fixed in marketing.

Iteration Checklist

  • Cardinal red appears exactly once per view (one CTA, one metric, one status pip)
  • All headings and buttons are sentence case — not Title Case
  • No gradients (text, buttons, backgrounds)
  • No emoji in any copy or label
  • Geist 600 with −0.025em tracking for display headings (28px+) — not body or UI labels
  • Geist Mono used only for IDs, codes, metadata, and short system labels
  • Card radius is 6–12px — no fully-rounded cards, no pill-shaped cards
  • Shadows, if present, use rgba(26,23,20,...) warm tint
  • No nested cards — use spacing and type hierarchy for structure
  • Bronze (#A67C52) used for editorial accents only (pull-quotes, dividers, medals)
